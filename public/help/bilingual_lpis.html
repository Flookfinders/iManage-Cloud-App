Overview

A bilingual gazetteer is identified in the corresponding Metadata  Record using the code BIL [Bilingual] in the Language field under Miscellaneous.

Property metadata

(England & Wales only)

To select your Property metadata, click on Property metadata under General in the Settings. You will see three boxes:

Click in the box to edit the data. Make the changes required and click Done when you are happy or Cancel to return without updating the details.

Gazetteer

Name

The name of the gazetteer e.g. LLPG for London Borough of Camden. Note: For Scotland this is a fixed list for selection.

Scope

A description of the content of the gazetteer e.g. all basic property units with some land parcels. Note: For Scotland this is "Scottish Gazetteer Conventions" and cannot be changed.

Territory

The geographic domain of the gazetteer.

Linked data

A list of the application datasets used to update the LLPG.

Owner

The organisation with overall responsibility for the gazetteer.

Frequency

The frequency with which the LLPG is maintained to the NLPG hub. Please select the appropriate option from the list.
Custodian

Name

The name for the Custodian

UPRN

The Unique Property Reference Number for the Custodian.

Code

The authority code for the Custodian
Miscellaneous

Classification scheme

This is the classification scheme used for all multiple value specified fields e.g. NLPG conventions document v1.0. Note: For Scotland this is "Scottish Gazetteer Conventions v4.0" and cannot be changed.

Metadata date

This is the date at which the gazetteer can be considered to be current.

Language

The default language used for the descriptors within the gazetteer. For Scottish customers the Language should be English if no Gaelic language records are present or Gaelic if the gazetteer contains any Gaelic language records. Please refer to the Bilingual Gazetteers topic for more information if required.

Character set

The description of the Character set used for the data when it is exported.

Content

Please use the slider for each entry to select the Percentage of each type of street that are present in your LSG.

If a street, or address, is provided within the Gazetteer, which is identified as being a bilingual gazetteer, it must be provided in both languages.

The Language for a Street is maintained in the Street Details and if the Metadata Language is set to Bilingual you will have two entries for each Street, one set to English and the other Welsh.

For example, a public building such as a library must be recorded in the LLPG as a BLPU and must have two Approved Preferred LPIs, one with the language flag set to English and the other to Welsh with the Addressable Object [PAO / SAO] either translated into or captured in the appropriate language. In some instances there is no difference between these values, for example property numbers, as no literal translation is either applicable or appropriate.

Any number of further Alternative or Historical LPIs can be held and flagged under the appropriate language. For example the Welsh speakers in the community may have a local descriptive name for the library but for each LPI created in one language a mirror version must be created in the other language, thus, there is a 1:1 relationship between each English and Welsh LPI.

The LPI keys of each related English and Welsh LPI are concatenated and held as a Cross Reference with a source code of nnnnBG [where nnnn is your authority code]. Thus, the Cross Reference for an English LPI with an LPI key of 6820L000000001 which has a 1:1 relationship with a Welsh LPI with an LPI key of 6820L000000002 is 6820L0000000016820L000000002.

If a BLPU has multiple LPI pairings more than one Cross Reference must exist i.e. one for each pair. This applies to records with any Logical Status.

How this works in iManage Cloud

In iManage Cloud we have tried to automate this process as much as possible:

•When you create a new LPI two LPIs will be created in the Property Form and the information that you enter into the first language will be carried forward to the second. When you click Save the Linked LPI [nnnnBG ] Cross Reference will be automatically created.

•When you change the Logical Status of an LPI the Logical Status of the linked Language LPI will also be changed.

•When you delete an LPI it will automatically delete the linked language LPI and the Linked LPI Cross Reference.

•When you delete the Linked LPI Cross Reference you will be asked if you want to archive the cross reference and the associated LPIs. Clicking Yes will remove both the Cross Reference and the linked LPIs.

•You will not be allowed to Add or Edit a Cross Reference with a Source of Linked LPI.
